Contribution Guideline
===========================================

Thank you for your interest in the Ploggify project!
This document describes the direction for contributing to the project, coding standards, and the Pull Request (PR) process.

How to Contribute
-------------------------------
We welcome all forms of contributions, including bug reports, feature suggestions, documentation improvements, and code contributions.

1. **Report Bugs**
   
    - Please report bugs using the **Issues** tab on GitHub.
    - Please describe the environment where the bug occurred (OS, browser, etc.) and how to reproduce it in detail.

2. **Suggest Features**
   
    - If you have an idea for a new feature, please create an Issue and attach the 'enhancement' label.

3. **Pull Request Process**
    1. **Create Branch:** Create a new branch from the 'main' branch before starting work.
    2. **Work and Commit:** Work on the branch and commit your changes.
    3. **Push:** Push the branch to your remote repository.
    4. **Create Pull Request:** Create a pull request on GitHub to merge your branch into the 'main' branch.
    5. **Review and Merge:** The maintainer will review the code and then proceed with the merge.

Contact
-------------------------------
If you have any questions regarding contribution, please contact us via GitHub Issue or email.
